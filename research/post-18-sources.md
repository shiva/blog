# Research: Post 18 — The Four-Memory-Type Problem: How SilicaDB Models Agent Cognition

## Key Sources
- CoALA Framework arXiv:2309.02427 — four types, SilicaDB maps to all four
- SODL spec (internal) — type hierarchy: MemoryRecord base, four subtypes
- arXiv:2602.19320: Anatomy of Agentic Memory — procedural memory via LEGOMem, TiMem
- arXiv:2606.24775: Are We Ready For An Agent-Native Memory System? — full taxonomy
- Atlan: Types of AI Agent Memory — working/episodic/semantic/procedural with enterprise extension

## Storyteller Tactic
**Hero & Guide** — SilicaDB as the guide who gives the developer-hero the tools to implement all four memory types correctly. Each type gets a concrete interface, not a vague API.

## Key Point
SilicaDB is the first embedded memory runtime to provide native support for all four CoALA memory types, each with appropriate storage semantics, retrieval behavior, and temporal validity enforcement.
